1.11 Subiendo al servidor remoto

EL siguiente comando le dirá a Git a donde quieres subir tus cambios cuando estén listos y, estamos listos para hacer una subida a nuestro servidor remoto (en GitHub).

Nuestro repositorio se llama origin y el branch por defecto se llama master. El modificador -u sirve para recordar los parametros para que, la próxima subida sea más sencilla y podas usar, directamente, git push sin ningún modificador. Escribe el siguiente comando y realiza una subida al servidor remoto:

git push -u origin master

1.11 Subiendo al servidor remoto

The push command tells Git where to put our commits when we're ready, and boy we're ready. So let's push our local changes to our origin repo (on GitHub).

The name of our remote is origin and the default local branch name is master. The -u tells Git to remember the parameters, so that next time we can simply run git push and Git will know what to do. Go ahead and push it!

git push -u origin master
